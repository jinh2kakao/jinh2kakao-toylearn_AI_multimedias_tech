## 프롬프트 유형과 선택 이유
1. [핵심 행동] 10. 재구성/번역 (Restructuring)
    선택 이유: 훈련 초점인 "복잡한 텍스트의 요약 및 매체 변환(Text-to-Script)"과 정확히 일치하는 유형입니다. "긴 보고서를 짧은 SNS 게시물로 요약"하는 예시처럼, 이 문제는 **"10페이지 PDF를 60초 JSON 스크립트로 재구성"**하는 것이 핵심 행동(Action)입니다.

    효과: AI에게 '요약'이 아닌 '변환'을 지시하여, 단순히 텍스트 길이를 줄이는 것이 아니라 영상이라는 새로운 포맷으로의 완전한 변형을 유도합니다.

2. [전문성 확보] 3. 역할 부여 (Role-Playing)
    선택 이유: "초등학생도 이해하는 수준" + "초반 3초 후킹" + "시각적 요소"는 일반적인 AI가 아닌 '숏폼 전문 PD'의 영역입니다. AI에게 "100만 뷰 숏폼 콘텐츠 PD" 또는 **"바이럴 지식 콘텐츠 작가"**라는 명확한 역할을 부여해야만, '영상 문법'에 맞는 창의적인 결과물을 얻을 수 있습니다.

    효과: "윤리적 쟁점은..."이라는 딱딱한 설명 대신, "AI가 내 생각을 훔쳐본다면?"과 같은 역할에 몰입한 '후킹 멘트'를 생성하게 만듭니다.

3. [복잡도 제어] 1. 지시형 (Instructional)
    선택 이유: "초등학생도 이해하는 수준으로"라는 요구는 매우 강력하고 구체적인 '지시'입니다. 이는 AI가 복잡한 원본 텍스트에 압도되지 않고, **'단순화'**라는 최우선 목표를 달성하도록 강제하는 역할을 합니다.

    효과: AI가 원본의 전문 용어(예: 알고리즘 편향성)를 버리고, "AI가 나쁜 말만 골라 배우면?"과 같은 쉬운 비유로 대체하도록 하는 강력한 지침이 됩니다.

4. [출력 통제] 4. 제약 조건 (Constraint-Based)
    선택 이유: 이 과제는 제약 조건의 집합체입니다. '60초 분량', '3초 후킹', 'JSON 형식', '시각적 요소(자막, 효과음) 포함' 등 모든 명확한 가이드라인을 AI에게 전달해야 합니다.

    효과: AI가 자유롭게 창작하다가 길을 잃는 것을 방지하고, PD(사용자)가 즉시 복사하여 사용할 수 있는, 실무에 완벽히 부합하는 '구조화된 데이터(JSON)'를 생성하게 만듭니다.

## 작성 위해 추가 필요 요건
이 프롬프트를 완벽하게 실행하기 위해 AI에게는 다음과 같은 핵심 입력 데이터가 반드시 필요합니다.

[필수] 10페이지 리포트 원본 텍스트:

변환의 대상이 되는 원본 소스입니다. 이 텍스트가 없으면 AI는 작업을 수행할 수 없습니다. 프롬프트 내에 [입력 텍스트] 섹션을 만들어 전체 내용을 붙여넣어야 합니다.

[선택] 채널의 기존 톤앤매너:

'테크인사이드' 채널이 유쾌한지, 진지한지, 감성적인지에 따라 스크립트의 어조가 달라집니다.

해결책: 이 정보가 없다면, 프롬프트에서 **"빠르고, 명료하며, 살짝 유쾌한 톤"**이라고 AI에게 톤앤매너를 가정하고 지정해 주는 것이 좋습니다.

## 가상 문제 상황
1. 가상 문제 상황
👤 페르소나: 뉴미디어 스타트업의 '숏폼 콘텐츠 PD'
🏙️ 배경: 당신은 IT 및 테크 트렌드를 소개하는 유튜브 채널을 운영하는 스타트업 '테크인사이드'의 PD입니다. 최근 회사는 틱톡과 릴스 유입을 늘리기 위해 '60초 지식 요약' 시리즈를 런칭했습니다.
🚨 핵심 문제: 마케팅 팀으로부터 **"생성형 AI의 미래와 윤리적 쟁점"**이라는 주제의 10페이지짜리 심층 분석 리포트(PDF)를 전달받았습니다. 내용은 매우 전문적이고 딱딱한 텍스트 위주여서, 이를 그대로 읽으면 시청 지속 시간이 3초 미만으로 떨어질 것이 확실시되는 상황입니다.
🎯 해결 과제:
복잡하고 긴 텍스트 리포트 내용을 대중이 이해하기 쉬운 '초등학생도 이해하는 수준'의 언어로 변환해야 합니다.
단순 요약을 넘어, 시각적 요소(화면 구성, 자막, 효과음)가 포함된 **60초 분량의 숏폼 영상 촬영 대본(Script)**을 완성해야 합니다. (특히, 초반 3초 안에 시선을 끄는 '후킹(Hooking)' 멘트가 필수입니다.)
출력은 JSON 형식
2. 훈련 초점 (힌트)
이 문제는 [복잡한 텍스트의 요약 및 매체 변환(Text-to-Script)] 능력을 평가
단순히 내용을 줄이는 것이 아니라, **영상 문법(Visual Cues)**에 맞게 내용을 재구성하는 프롬프트 엔지니어링이 필요.
